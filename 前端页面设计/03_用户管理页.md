# 页面名称：用户管理页

> **给 AI 的指令**：
>
> 请根据《项目全局技术栈与开发规范》以及下方的页面需求文档，为我编写 Vue 3 + Vite + TypeScript 前端代码。要求逻辑清晰、组件拆分合理，并使用严格的 TypeScript 类型。

## 1. 页面基本信息

- **页面路由 (Route)**：`/admin/users`
- **页面角色权限**：仅管理员访问
- **核心目标**：管理员创建、编辑、禁用和删除内部用户账号。

## 2. 页面布局与视觉结构 (Layout & Visuals)

本页面分为以下核心区域：

1. **顶部操作区 (Toolbar)**：
   - 左侧：关键词输入框、角色筛选、状态筛选。
   - 右侧：“新增用户”按钮。
2. **中间数据区 (Table)**：
   - 列：ID、邮箱、昵称、角色、状态、每日请求上限、每日成本上限、创建时间、操作。
   - 操作：编辑、禁用/启用、删除。
3. **弹窗区 (Modal)**：
   - 新增/编辑用户表单。
   - 表单项：邮箱、昵称、角色、密码、状态、每日请求上限、每日成本上限。

## 3. 数据状态定义 (State Management)

- `tableData` (Array<UserItem>): 用户列表
- `isLoading` (Boolean): 表格加载状态
- `searchParams` (Object): `keyword`、`role`、`status`
- `pagination` (Object): `page`、`pageSize`、`total`
- `modalVisible` (Boolean): 弹窗状态
- `modalType` (String): `'create' | 'update'`
- `currentRecord` (UserItem | null): 当前编辑用户

## 4. 交互逻辑与事件 (Interactions & Events)

1. 页面挂载时调用 `GET /api/v1/admin/users`。
2. 点击搜索或分页切换时重新请求列表。
3. 点击新增打开空表单。
4. 点击编辑打开当前用户表单。
5. 提交新增调用 `POST /api/v1/admin/users`。
6. 提交编辑调用 `PUT /api/v1/admin/users/:id`。
7. 删除前弹出确认框，确认后调用 `DELETE /api/v1/admin/users/:id`。

## 5. Mock 数据与接口结构 (Data Interface)

```ts
interface UserItem {
  id: number
  email: string
  display_name: string
  role: 'admin' | 'member'
  status: 'active' | 'disabled'
  daily_request_limit: number | null
  daily_cost_limit_micro_usd: number | null
  created_at: string
}
```

## 6. 组件拆分要求 (Component Splitting Rules)

1. 页面主入口：`views/admin/UsersView.vue`
2. 搜索区：`components/users/UserToolbar.vue`
3. 表格：`components/users/UserTable.vue`
4. 表单弹窗：`components/users/UserFormModal.vue`
