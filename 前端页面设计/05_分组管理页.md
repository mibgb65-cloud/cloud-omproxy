# 页面名称：分组管理页

> **给 AI 的指令**：
>
> 请根据《项目全局技术栈与开发规范》以及下方的页面需求文档，为我编写 Vue 3 + Vite + TypeScript 前端代码。要求逻辑清晰、组件拆分合理，并使用严格的 TypeScript 类型。

## 1. 页面基本信息

- **页面路由 (Route)**：`/admin/groups`
- **页面角色权限**：仅管理员访问
- **核心目标**：管理 API Key 和上游账号调度所需的分组。

## 2. 页面布局与视觉结构 (Layout & Visuals)

1. **顶部操作区**：
   - 状态筛选。
   - “新增分组”按钮。
2. **分组表格**：
   - 列：ID、名称、描述、默认平台、状态、排序、操作。
3. **表单弹窗**：
   - 表单项：名称、描述、默认平台、状态、排序。

## 3. 数据状态定义 (State Management)

- `groupList` (Array<GroupItem>)
- `isLoading` (Boolean)
- `statusFilter` (String)
- `modalVisible` (Boolean)
- `modalType` (`'create' | 'update'`)
- `currentRecord` (GroupItem | null)

## 4. 交互逻辑与事件 (Interactions & Events)

1. 页面挂载时调用 `GET /api/v1/admin/groups`。
2. 新增分组调用 `POST /api/v1/admin/groups`。
3. 编辑分组调用 `PUT /api/v1/admin/groups/:id`。
4. 删除分组调用 `DELETE /api/v1/admin/groups/:id`。
5. 如果后端返回分组被使用的错误，弹出明确提示。

## 5. Mock 数据与接口结构 (Data Interface)

```ts
interface GroupItem {
  id: number
  name: string
  description: string | null
  default_platform: 'anthropic' | 'openai' | 'gemini' | null
  status: 'active' | 'disabled'
  sort_order: number
}
```

## 6. 组件拆分要求 (Component Splitting Rules)

1. 页面主入口：`views/admin/GroupsView.vue`
2. 操作区：`components/groups/GroupToolbar.vue`
3. 表格：`components/groups/GroupTable.vue`
4. 表单弹窗：`components/groups/GroupFormModal.vue`
