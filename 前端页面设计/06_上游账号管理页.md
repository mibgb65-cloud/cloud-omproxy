# 页面名称：上游账号管理页

> **给 AI 的指令**：
>
> 请根据《项目全局技术栈与开发规范》以及下方的页面需求文档，为我编写 Vue 3 + Vite + TypeScript 前端代码。要求逻辑清晰、组件拆分合理，并使用严格的 TypeScript 类型。

## 1. 页面基本信息

- **页面路由 (Route)**：`/admin/upstream-accounts`
- **页面角色权限**：仅管理员访问
- **核心目标**：管理 Anthropic、OpenAI、Gemini 上游账号和分组绑定。

## 2. 页面布局与视觉结构 (Layout & Visuals)

1. **顶部操作区**：
   - 平台筛选、状态筛选、分组筛选。
   - “新增上游账号”按钮。
2. **账号表格**：
   - 列：ID、名称、平台、认证方式、分组、优先级、状态、冷却时间、最近使用、操作。
   - 操作：测试、编辑、禁用/启用、删除。
3. **账号表单弹窗**：
   - 名称、平台、认证方式、凭证、Base URL、优先级、分组、多选状态。
   - 凭证字段编辑时只允许输入新值，不显示旧值。
4. **测试结果弹窗**：
   - 显示测试成功/失败、耗时、错误信息。

## 3. 数据状态定义 (State Management)

- `tableData` (Array<UpstreamAccountItem>)
- `groups` (Array<GroupOption>)
- `isLoading` (Boolean)
- `testingAccountId` (Number | null)
- `searchParams` (Object): `platform`、`status`、`group_id`
- `modalVisible` (Boolean)
- `currentRecord` (UpstreamAccountItem | null)

## 4. 交互逻辑与事件 (Interactions & Events)

1. 页面挂载时加载账号列表和分组选项。
2. 新增调用 `POST /api/v1/admin/upstream-accounts`。
3. 编辑调用 `PUT /api/v1/admin/upstream-accounts/:id`。
4. 删除调用 `DELETE /api/v1/admin/upstream-accounts/:id`。
5. 点击测试调用 `POST /api/v1/admin/upstream-accounts/:id/test`。
6. 所有保存操作成功后刷新列表。

## 5. Mock 数据与接口结构 (Data Interface)

```ts
interface UpstreamAccountItem {
  id: number
  name: string
  platform: 'anthropic' | 'openai' | 'gemini'
  auth_type: 'api_key' | 'oauth'
  status: 'active' | 'disabled' | 'error'
  priority: number
  groups: Array<{ id: number; name: string }>
  cooldown_until: string | null
  last_used_at: string | null
}
```

4. 表单弹窗：`components/upstream/UpstreamAccountFormModal.vue`
5. 测试结果弹窗：`components/upstream/AccountTestResultModal.vue`
