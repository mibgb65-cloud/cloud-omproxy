# 页面名称：用量记录页

> **给 AI 的指令**：
>
> 请根据《项目全局技术栈与开发规范》以及下方的页面需求文档，为我编写 Vue 3 + Vite + TypeScript 前端代码。要求逻辑清晰、组件拆分合理，并使用严格的 TypeScript 类型。

## 1. 页面基本信息

- **页面路由 (Route)**：`/admin/usage-logs`
- **页面角色权限**：仅管理员访问
- **核心目标**：查看内部用户的请求明细、token、模型和估算成本。

## 2. 页面布局与视觉结构 (Layout & Visuals)

1. **统计摘要区**：
   - 当前筛选范围内请求数、token、估算成本。
2. **筛选区**：
   - 时间范围、用户、API Key、平台、模型。
   - 查询按钮和重置按钮。
3. **日志表格区**：
   - 列：时间、用户、API Key、平台、模型、endpoint、token、估算成本、状态码、耗时、错误。
4. **详情抽屉**：
   - 展示单条请求的完整字段和错误摘要。

## 3. 数据状态定义 (State Management)

- `stats` (Object | null): 当前筛选统计
- `tableData` (Array<UsageLogItem>)
- `isLoading` (Boolean)
- `searchParams` (Object): 时间、用户、平台、模型等
- `pagination` (Object)
- `detailVisible` (Boolean)
- `currentLog` (UsageLogItem | null)

## 4. 交互逻辑与事件 (Interactions & Events)

1. 页面挂载时默认查询今天的数据。
2. 查询按钮调用 `GET /api/v1/admin/usage-logs` 和 `GET /api/v1/admin/usage-stats`。
3. 重置按钮清空筛选并回到今天。
4. 点击表格行打开详情抽屉。
5. 分页变化时保持筛选条件并重新请求。

## 5. Mock 数据与接口结构 (Data Interface)

```ts
interface UsageLogItem {
  id: number
  request_id: string
  user_id: number
  platform: string
  endpoint: string
  model: string | null
  input_tokens: number
  output_tokens: number
  estimated_cost_micro_usd: number
  status_code: number
  duration_ms: number | null
  is_stream: boolean
  error_message: string | null
  created_at: string
}
```

## 6. 组件拆分要求 (Component Splitting Rules)

1. 页面主入口：`views/admin/UsageLogsView.vue`
2. 统计摘要：`components/usage/UsageSummaryCards.vue`
3. 筛选区：`components/usage/UsageFilterBar.vue`
4. 表格：`components/usage/UsageLogTable.vue`
5. 详情抽屉：`components/usage/UsageLogDrawer.vue`
