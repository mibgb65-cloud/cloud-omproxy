# 页面名称：设置页

> **给 AI 的指令**：
>
> 请根据《项目全局技术栈与开发规范》以及下方的页面需求文档，为我编写 Vue 3 + Vite + TypeScript 前端代码。要求逻辑清晰、组件拆分合理，并使用严格的 TypeScript 类型。

## 1. 页面基本信息

- **页面路由 (Route)**：`/admin/settings`
- **页面角色权限**：仅管理员访问
- **核心目标**：配置站点信息、网关默认设置、备份设置和模型价格。

## 2. 页面布局与视觉结构 (Layout & Visuals)

1. **站点设置区**：
   - 站点名称。
2. **网关设置区**：
   - 默认分组。
   - 是否启用简单每日上限。
3. **备份设置区**：
   - R2 备份启用状态。
   - 手动备份入口跳转。
4. **模型价格区**：
   - 表格展示平台、模型、输入单价、输出单价、缓存单价、状态。
   - 支持编辑价格。

## 3. 数据状态定义 (State Management)

- `settings` (Object | null)
- `modelPrices` (Array<ModelPriceItem>)
- `groups` (Array<GroupOption>)
- `isLoading` (Boolean)
- `isSaving` (Boolean)
- `priceModalVisible` (Boolean)
- `currentPrice` (ModelPriceItem | null)

## 4. 交互逻辑与事件 (Interactions & Events)

1. 页面挂载时加载 `GET /api/v1/admin/settings`、`GET /api/v1/admin/model-prices`、分组选项。
2. 点击保存设置调用 `PUT /api/v1/admin/settings`。
3. 点击编辑模型价格打开弹窗。
4. 保存价格调用 `PUT /api/v1/admin/model-prices/:id`。
5. 保存成功后刷新当前设置或价格列表。

## 5. Mock 数据与接口结构 (Data Interface)

```ts
interface ModelPriceItem {
  id: number
  platform: 'anthropic' | 'openai' | 'gemini'
  model: string
  input_price_micro_usd_per_token: number
  output_price_micro_usd_per_token: number
  cache_read_price_micro_usd_per_token: number
  cache_write_price_micro_usd_per_token: number
  status: 'active' | 'disabled'
}
```

## 6. 组件拆分要求 (Component Splitting Rules)

1. 页面主入口：`views/admin/SettingsView.vue`
2. 站点设置表单：`components/settings/SiteSettingsForm.vue`
3. 网关设置表单：`components/settings/GatewaySettingsForm.vue`
4. 模型价格表格：`components/settings/ModelPriceTable.vue`
5. 价格编辑弹窗：`components/settings/ModelPriceModal.vue`
