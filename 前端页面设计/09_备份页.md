# 页面名称：备份页

> **给 AI 的指令**：
>
> 请根据《项目全局技术栈与开发规范》以及下方的页面需求文档，为我编写 Vue 3 + Vite + TypeScript 前端代码。要求逻辑清晰、组件拆分合理，并使用严格的 TypeScript 类型。

## 1. 页面基本信息

- **页面路由 (Route)**：`/admin/backups`
- **页面角色权限**：仅管理员访问
- **核心目标**：创建手动备份、查看 R2 备份记录并下载备份文件。

## 2. 页面布局与视觉结构 (Layout & Visuals)

1. **顶部说明区**：
   - 简短说明 R2 备份用途。
   - 提醒敏感字段会加密或脱敏。
2. **操作区**：
   - “创建手动备份”按钮。
   - `include_usage_logs` 开关。
3. **备份列表区**：
   - 列：ID、文件名、类型、大小、状态、创建时间、完成时间、操作。
   - 操作：下载。
4. **状态提示区**：
   - 创建中 loading。
   - 失败时展示错误信息。

## 3. 数据状态定义 (State Management)

- `backupList` (Array<BackupItem>)
- `isLoading` (Boolean)
- `isCreating` (Boolean)
- `includeUsageLogs` (Boolean)
- `pagination` (Object)

## 4. 交互逻辑与事件 (Interactions & Events)

1. 页面挂载时调用 `GET /api/v1/admin/backups`。
2. 点击创建手动备份：
   - 弹出确认框。
   - 调用 `POST /api/v1/admin/backups`。
   - 成功后刷新列表。
3. 点击下载：
   - 调用 `GET /api/v1/admin/backups/:id/download-url`。
   - 使用返回的 `download_url` 打开新窗口下载。
4. 分页切换时重新加载列表。

## 5. Mock 数据与接口结构 (Data Interface)

```ts
interface BackupItem {
  id: number
  backup_type: 'manual' | 'scheduled' | 'usage_export'
  file_name: string
  file_size: number
  status: 'pending' | 'completed' | 'failed'
  created_at: string
  completed_at: string | null
  error_message?: string | null
}
```

3. 表格：`components/backups/BackupTable.vue`
4. 创建确认弹窗可复用全局确认组件。
