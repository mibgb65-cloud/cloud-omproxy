# 表名：`orders` (订单表)

## 1. 基本信息
- **表描述**：存储用户的订单基础信息。
- **字符集**：`utf8mb4`
- **排序规则**：`utf8mb4_unicode_ci`
- **核心业务逻辑**：一个用户可以有多个订单，订单金额以“分”为单位存储以避免精度问题。

## 2. 字段定义 (Schema)

| 字段名 (Field) | 类型 (Type) | 必填 (NN) | 默认值 (Default)  | 注释与业务规则 (Comment/Rules)                             |
| :------------- | :---------- | :-------- | :---------------- | :--------------------------------------------------------- |
| `id`           | BIGINT      | 是        | 自增              | 主键 (Primary Key)                                         |
| `order_no`     | VARCHAR(64) | 是        | 无                | 订单流水号，业务全局唯一                                   |
| `user_id`      | BIGINT      | 是        | 无                | 关联 `users.id`，买家ID                                    |
| `total_amount` | INT         | 是        | 0                 | 订单总金额（单位：分），不能为负数                         |
| `status`       | TINYINT     | 是        | 1                 | 订单状态：1=待支付, 2=已支付, 3=已发货, 4=已完成, 5=已取消 |
| `pay_time`     | DATETIME    | 否        | NULL              | 支付成功的时间                                             |
| `is_deleted`   | TINYINT     | 是        | 0                 | 软删除标识：0=正常, 1=已删除                               |
| `created_at`   | DATETIME    | 是        | CURRENT_TIMESTAMP | 记录创建时间                                               |
| `updated_at`   | DATETIME    | 是        | CURRENT_TIMESTAMP | 记录更新时间，随行更新                                     |

## 3. 索引定义 (Indexes)

- **主键索引**：`id`
- **唯一索引 (Unique)**：`uk_order_no` (`order_no`) - 确保订单号不重复。
- **普通索引 (Normal)**：`idx_user_id` (`user_id`) - 加速按用户查询订单的列表接口。
- **复合索引 (Composite)**：`idx_user_status` (`user_id`, `status`) - 用于“查询某用户特定状态订单”的常见场景。

## 4. 关联关系与特殊说明 (Relations & Notes)
1. **外键说明**：本表 `user_id` 逻辑关联 `users` 表的 `id`，但**不在数据库层面建立物理外键**，由程序代码保障一致性。
2. **软删除规范**：所有查询必须默认带上 `is_deleted = 0` 的条件。
3. **金额计算**：凡是涉及 `total_amount` 的计算，请在代码层面注意单位的转换（前端展示为元，存入数据库为分）。
