# 接口名称：[例如：创建新订单]

## 1. 基本信息
- **路径 (Path)**：`/api/v1/orders`
- **方法 (Method)**：`POST`
- **功能描述**：用户在购物车点击结算后，生成待支付订单。
- **前置条件**：需携带有效的 User Token (JWT)。

## 2. 请求参数 (Request)
**Header:**
- `Authorization`: Bearer <token>

**Body (JSON):**
- `product_id` (String, 必填): 商品 ID
- `quantity` (Integer, 必填): 购买数量，必须 > 0
- `coupon_code` (String, 选填): 优惠券码

## 3. 核心处理逻辑 (Step-by-Step)
请按照以下步骤严格实现业务逻辑：

1. **参数校验**：
   - 检查 `quantity` 是否大于 0，否则抛出 `400 Bad Request`，提示“数量必须大于0”。
2. **获取用户信息**：
   - 解析 Header 中的 Token，获取 `user_id`。
3. **查询商品与库存**：
   - 查 `products` 表获取商品信息。
   - 判断 `products.stock` 是否 >= `quantity`。
   - 【异常分支】：如果库存不足，抛出 `409 Conflict`，提示“库存不足”。
4. **计算金额**：
   - 基础金额 = `products.price` * `quantity`。
   - 判断是否传入 `coupon_code`：
     - 如果有：查询 `coupons` 表验证有效性。若有效，扣减对应金额；若无效，抛出错误“无效的优惠券”。
5. **开启数据库事务 (Transaction)**：
   - a. 扣减库存：`UPDATE products SET stock = stock - {quantity} WHERE id = {product_id}`
   - b. 标记优惠券已使用（如果使用了）
   - c. 创建订单记录：插入 `orders` 表，状态设为 `pending_payment`。
   - 【异常分支】：如果有任何一步失败，回滚事务 (Rollback)。
6. **返回结果**：组装订单信息并返回 200 响应。

## 4. 响应格式 (Response)
**成功响应 (200 OK):**
{
  "code": 0,
  "data": {
    "order_id": "string",
    "total_amount": "decimal",
    "status": "pending_payment"
  }
}

**错误响应 (示例):**
{
  "code": 4001,
  "message": "库存不足"
}
